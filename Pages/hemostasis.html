PROCEPT BioRobotics - Hemostasis

Project Overview

PROCEPT BioRobotics is a medical device company that specializes in the treatment of Benign Prostatic Hyperplasia using a method known as Aquablation. After the Aquablation procedure is complete, many patients experience excessive bleeding. So, as a possible solution to help stop the bleeding, the R&D team designed a device that can be used to cauterize tissue during the procedure. Our task was to integrate the new feature into our main software coined as Hemostasis.

The Problem

A new device has been added to the main product, integrate new features into the software which can be utilized by the users during procedures.

Challenges

Designing a new feature for a software becomes a lot more difficult if it's extremely dependant on the hardware. Both the device used for cauterization and the new feature in the software were being designed at the same time, so if the device needs changes, the software will most likely have to change as well.

The timeline for this first iteration is short, so the new features should utilize components that already exist in the software. This would save the developers' time.

Understanding the Product

The Users

With the assistance from the engineering, marketing and software development teams, we were able to gather research on the current users of the product. We were easily able to determine main pain points the users might have when using our product.

The Current Software

The main software is divided into three panels: the left panel which usually contains instructional information as well as the buttons to proceed forward and back in the procedure; the middle panel which shows the ultrasound and TRUS data; and the right panel which shows all the steps and buttons to control the device.

Planning the Design

Informational Architecture

Luckily, we were easily able to integrate the new steps at the end of planning and treatment phase. The treatment phase was originially one step, but for the new design it was split up into two steps: Hemostasis and Cauterize.

For the new hemostasis treatment, after completing the Aquablation step, there will be a third new option to start the automated hemostasis treatment. After the hemostasis treatment is complete, the user will be brought back to the "pass complete" screen.

Wireframes

There were four screens that needed to be added in the new software: two in the planning phase and two during the treatment phase. We reused many of the components that already existed in the software.

Final Design

High Fidelity Prototypes

We utilized Invision Studios to create prototypes that were presented to stake holders. We designed the animation for the collapsible tabs as well as the hover states for the selectable sections. Lastly, we also designed the ability to adjust the angle for each section and the transition from the decision screen to the treatment phase.

Collapsible Tabs

Use the collapsible tables to return to previous steps.

Select an Angle Section

While in Sagittal view, select a section to enter the Transverse view of that section.

Customize the Angle

Use this screen to adjust the power level, change the angle or turn off the section.

Start Treatment

After completing Aquablation treatment, continue onto Hemostasis treatment.

Final Takeaway

Designing for a medical device company proved to be a lot different than designing most products. The users were narrowed down to a very specific group, in our case it was urologist, which can set limits to the amount of research we can gather.

When designing a product that is highly reliant on a hardware component, it's very important that the mechanical engineers, software developers and UI designers work very closely to plan it. The mechanical engineers need to make sure the UI designers understand what features should be included in order to control the hardware. The UI Designers need to work with the software developers to understand limitations of the software and pass along the design requiremnts. Errors can occur if there is a gap of knowledge between these teams.
