PROCEPT BioRobotics - Peripheral Simulator

Project Overview

PROCEPT's Robotic System consists of different components: the CPU, the Console, and the Motorpack/Handpiece. The software development team created a program called the console simulator in order to mimic the Console which then can be used in R&D testing. My task was to redesign the original program so that it includes some the other new peripheral simulators as well as improve usability.

The Problem

There will new simulators that the software development team will need for testing in the future, redesign the program so it can include the new peripherals and improve the layout so that it's easy to learn to use.

Challenges

This project was pretty straight forward and since there was an existing software there was a lot to work off. The main challenge is that there was only going to be a handful of users utilizing the software, so the feedback will be minimal.

Understanding the Product

The Users

This peripheral software was created for the hardware engineers to use for testing. So, it was easy to get the design requirements and feedback since they were part of the team.

The Current Software

The simulator's main purpose was to send data to the CPU, which contains the main software used in the procedure, and then display returned data. The old software design helped give us the requirements for the new software.

Planning the Design

Informational Architecture

The Console simulator had two sections, Test and Data. Each section will have it's own menu bar since the view and presets options will be different in both sections. Settings and recording will remain the same for both sections. The Test section also has three sections within for the currently available configurations. The Data section will mainly display a table with the data sets with options for customization.

Low-Fidelity Prototypes

Tabs were used in order to create hierarchy. The buttons to cycle through the peripherals are on left and the components for them will be on the right. There are two main screens, Test and Data.

Final Design

Using Invision Studios, we were able to create high fidelity prototypes of a design that is currently in development.

High Fidelity Prototypes

For colors and fonts, we followed the company's style guide. A darker pallete was used in order to match the main software's dark color scheme. The company's primary color was used for a lot of the main buttons. Calibri was used as a the main font as required by the style guide.

Send Out Data

Use the Test Tab to send data to the CPU.

Receive Incoming Data

Use the Data tab to display data sets and customize the items displayed.

Final Takeaway

One of the biggest advantages of designing the software is being able to receive user feedback quickly, especially since the users were part of the development team. Although, we didn't utilize proper methods for feedback from the user which could've greatly helped in creating a better design. The initial design that the software team developed was also very useful for deciding what components the users needed. We didn't have to conduct too much research on what features were needed since they were laid out explicitly.
